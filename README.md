## 为什么你会想用它

你可能已经试过很多 AI 写作工具：
- 一开始很惊艳，写到第 3 章就崩；
- 人设前后打架，剧情像失忆；
- 想“微调一段”，结果整章重写；
- 导不出、存不住、换模型就推倒重来。

**墨渊** 的目标很直接：

✅ **把“偶尔写得好”变成“稳定写下去”**。  
✅ **把“灵感爆发”变成“可复用流程”**。  
✅ **把“单次生成”变成“完整作品生产线”**。

---

## 它到底强在哪（核心卖点）

### 1) 面向“长篇完结”的工作流，而不是一次性生成
从立项设定（书名/题材/角色/世界观/主线）到章节管理，再到正文生成、局部修改和最终导出，全部在同一个界面闭环完成。

### 2) 章节连续性有策略，不靠运气
生成当前章节时会携带前文关键信息（前情摘要 + 上章结尾），减少人物失真和剧情断层。

### 3) 支持“局部精修”，不必每次推倒重写
针对单章做多轮定向修改：节奏、措辞、张力、情绪线都可逐步打磨，更贴近真实写作过程。

### 4) 多模型自由切换，成本与质量自己掌控
内置 DeepSeek / OpenAI / Claude，并支持自定义兼容端点。你可以按预算、速度、风格偏好灵活选择。

### 5) 纯前端、零后端、开箱即用
一个 `index.html` 就能跑，本地保存进度，不需要数据库和服务端部署。

## 功能全景

- **项目设定**：书名、题材、风格、人物、世界观、核心剧情。
- **章节编排**：章节数量、标题、章节概要、状态管理。
- **正文生成**：按章节一键生成，支持流式实时输出。
- **章节改写**：局部修改指令，多轮精修。
- **写作存档**：统计已完成章节与字数。
- **全文导出**：一键导出 `.md` / `.txt`。
- **配置中心**：模型供应商、API Key、温度、最大 token、流式开关、连接测试。

## 3 分钟上手

### 1. 启动项目

```bash
git clone <your-repo-url>
cd novel
python3 -m http.server 8000
```

打开 `http://localhost:8000`。

> 也可直接双击 `index.html` 打开，但推荐使用本地静态服务以获得更稳定的下载/复制体验。

### 2. 配置模型

1. 进入「设置」。
2. 选择 provider（DeepSeek / OpenAI / Claude / Custom）。
3. 填写 API Key；若为 Custom 再填写 Endpoint 与模型名。
4. 点击“连接测试”确认可用。

### 3. 开始创作

1. 在「设定」页完成人物与剧情主线。  
2. 在「章节」页补全每章概要。  
3. 在「写作」页逐章生成并精修。  
4. 在「存档」页导出全书。

## 写出“能看、能连载、能完结”的实战建议

- **把大纲写成冲突链，而不是概念描述**：谁要什么、谁阻止、代价是什么。
- **每章先写“目标—阻碍—结果”三点**：可显著降低跑题。
- **大改走“章节重生”，小改走“局部修改”**：效率最高。
- **关键转折章先导出备份**：保证可回退、可对比。

## 数据与安全

- 项目内容与配置默认保存在浏览器本地（`localStorage`）。
- API Key 是否长期保存由“持久记住 API Key”开关控制。
- 建议不要在公共或不可信设备中启用 Key 持久化。
- 本项目不内置后端转发，模型请求由浏览器直接发往目标 API。
